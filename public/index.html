Usage Guide

Dashboard

The KPI cards at the top show totals for servers, online/offline status, pending updates, and more. Click any card to see a detailed list of the servers behind that number.

Servers

Browse or search all servers. Click a row to open the server detail popup with tabs for Network, Disks, Roles, Services, Local Admins, Missing Updates, Event Errors, Firewall, Local Policies, and Logbook.

Maintenance

View all servers currently in maintenance mode. You can set or end maintenance from the server detail popup — a banner at the top of the popup shows the current state.

Logbook

A chronological log of notes across all servers. Add entries directly from the server detail popup using the Add Logbook Entry button at the top.

Domains

Switch between GPO and DNS sub-tabs to review Group Policy Objects and DNS zones collected from domain controllers.

Script Deployment

Collect-ServerInventory.ps1

The PowerShell collection script gathers OS info, hardware, network, disks, patches, services, roles, security status, firewall rules, GPOs, local policies, event logs, and more. All operations are read-only.

Parameters

.\Collect-ServerInventory.ps1 -ApiUrl "https://<vire-host>/api/inventory" -ApiKey "<your-api-key>"

Deployment as Scheduled Task

  1. Copy Collect-ServerInventory.ps1 to each server (e.g. C:\Scripts\).
  2. Create a Scheduled Task running as SYSTEM.
  3. Set the action to:
    powershell.exe -ExecutionPolicy Bypass -File "C:\Scripts\Collect-ServerInventory.ps1" -ApiUrl "https://<vire-host>/api/inventory" -ApiKey "<your-api-key>"
  4. Set the trigger to run every 4–6 hours.

GPO Deployment (recommended)

Use a Group Policy Preference Scheduled Task to push the script and schedule to all servers at once — no manual per-server setup needed.

Send-Heartbeat.ps1

Lightweight script that sends CPU, RAM, disk, and stopped-service status every minute. The server must already be registered via the full inventory script.

.\Send-Heartbeat.ps1 -ApiUrl "https://<vire-host>" -ApiKey "<your-api-key>"

Run as SYSTEM via Scheduled Task, triggered every 1 minute. Resource impact: ~5 MB RAM, <200ms CPU per run.

Collect-DomainGPO.ps1

Collects all Group Policy Objects and DNS zones from Active Directory. Must be run on a Domain Controller or a machine with the GroupPolicy and DnsServer modules (RSAT).

.\Collect-DomainGPO.ps1 -ApiUrl "https://<vire-host>" -ApiKey "<your-api-key>"

Run as a Domain Admin (or an account with GPO read permissions). Recommended frequency: every 4–6 hours. Optionally specify -Domain "contoso.com" to target a specific domain.

API Key

The API key is configured in settings.json on the Vire server under the apiKey field.
